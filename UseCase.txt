Accesso alla piattaforma
L’utente già registrato accede alla piattaforma inserendo le proprie credenziali. Il sistema verifica la correttezza dei dati e, in caso positivo, consente l’accesso alle funzionalità disponibili in base al ruolo e al profilo dell’utente.

Acquisto di uno o più articoli
L’acquirente, dalla sua home page, può cercare gli articoli disponibili sulla piattaforma tramite una searchbar che produce dei risultati di ricerca globali, oppure visualizzare degli articoli disponibili direttamente nel feed verticale sottostante alla searchbar. Una volta trovato quello di suo gradimento, per procedere all’acquisto dovrà premere sull’articolo che desidera, che aprirà la scheda dettagliata
di quest’ultimo. In questa scheda è presente il pulsante per l’aggiunta al carrello. Una volta aggiunto l’ articolo al carrello, potrà accedere al carrello andando sul pulsante sempre presente sulla
dashboard in alto a destra. Nella schermata del carrello sarà possibile visualizzare gli articoli aggiunti e, per ognuno, in che quantità. La quantità potrà essere modificata tramite appositi pulsanti “+” e “-”. Una volta appurato ciò che si vuole acquistare dovrà premere sul pulsante “acquista” per completare l’acquisto inserendo per prima cosa i dati di spedizione, ovvero: Nome, Cognome,
Indirizzo, Numero Civico, CAP e Città. Fatto ciò procederà al pagamento inserendo i dati della carta di credito tramite un servizio di pagamento di terze parti. Infine riceve una notifica di avvenuto acquisto.

Distributore di Tipicità, Produttore, Trasformatore


Scenarios
Scenario
1.  L'Acquirente accede al carrello tramite l'icona nella dashboard
2.  SYSTEM Il sistema mostra il contenuto del carrello con gli articoli selezionati
3.  L'Acquirente conferma l'intenzione di procedere all'acquisto
4. Il sistema richiede l'inserimento dei dati di spedizione (Nome, Cognome, Indirizzo, Numero Civico, CAP, Città)
5.  L'Acquirente inserisce i dati di spedizione
6.  SYSTEM Il sistema reindirizza l'acquirente al sistema di pagamento di terze parti
7.  L'Acquirente inserisce i dati di pagamento
8.  Il sistema di pagamento processa la transazione
9.  SYSTEM Il sistema conferma l'avvenuto acquisto
10.  SYSTEM Il sistema reindirizza l'acquirente alla pagina degli ordini
Extensions
3.a. Modifica delle quantità di articoli nel carrello
1.  L'Acquirente modifica la quantità di un articolo nel carrello usando i pulsanti "+" o "-"
2.  Il sistema aggiorna il prezzo totale del carrello
3.  Il flusso riprende dal punto 6 della sequenza principale
3.b. Rimozione di articoli dal carrello
1.  L'Acquirente rimuove un articolo dal carrello
2.  Il sistema aggiorna il contenuto e il prezzo totale del carrello
3.  Se il carrello è vuoto, il sistema mostra un messaggio e reindirizza l'Acquirente al catalogo
4.  Altrimenti, il flusso riprende dal punto 6 della sequenza principale
8.a. Pagamento rifiutato per fondi insufficienti
1.  Il sistema di pagamento rileva fondi insufficienti
2.  Il sistema di pagamento notifica il rifiuto della transazione
3.  Il sistema mostra un messaggio di errore all'acquirente
4.  L'Acquirente conferma la ricezione del messaggio
5.  Il sistema reindirizza l'Acquirente al carrello
8.b. Malfunzionamento del sistema di pagamento
1.  Il sistema di pagamento riscontra un errore durante la transazione
2.  Il sistema notifica l'acquirente del problema con un messaggio di errore
3.  L'Acquirente conferma la ricezione del messaggio
4.  Il sistema reindirizza l'Acquirente al carrello, mantenendo gli articoli selezionati
8.c. Annullamento dell'acquisto
1.  L'Acquirente clicca sul pulsante "Annulla" in qualsiasi fase del processo di acquisto
2.  Il sistema chiede conferma dell'annullamento
3.  Se confermato, il sistema annulla il processo di acquisto
4.  Il sistema reindirizza l'Acquirente al carrello, mantenendo gli articoli selezionati

4.  SYSTEM Il sistema visualizza il modulo per l'inserimento delle informazioni dell'articolo.
5.  Il Venditore inserisce le informazioni richieste per ciascun campo del modulo.
6.  SYSTEM Il sistema verifica la validità dei dati inseriti per ogni campo.
7.  Il Venditore clicca sul pulsante "Salva" per confermare l'aggiunta dell'articolo.
8.  SYSTEM Il sistema salva le informazioni dell'articolo nel database.
9.  SYSTEM Il sistema invia l'articolo al Curatore per l'approvazione.

4.  Il caso d'uso termina.

1.  SYSTEM Il caricamento dell'articolo fallisce per un errore interno del sistema.
2.  SYSTEM Il sistema notifica il Venditore del problema.
3.  SYSTEM Il sistema reindirizza il Venditore alla Sezione Articoli.

2.  SYSTEM Il sistema invia una notifica al Venditore le motivazioni.

Caricamento di prodotti o materie prime
Il Distributore di Tipicità ha la possibilità di caricare sia materie prime che prodotti trasformati e dunque avrà la possibilità di indicare tramite un pulsante che tipo di risorsa caricare, se una materia prima o un prodotto trasformato, prima di passare alla specifica delle informazioni. Per quanto riguarda il caricamento di materie prime, il processo è analogo a quello del Produttore, dunque:


Tramite la propria dashboard personale, può caricare i prodotti desiderati specificando: nome, descrizione, immagine/i e prezzo. Successivamente, in un’altra sezione denominata “Informazioni sul prodotto” e composta dalle sottosezioni “Certificazioni” e “Metodi di coltivazione”, sarà possibile aggiungere le certificazioni (DOP, IGP e GlobalG.A.P.) in formato PDF e i metodi di coltivazione (Biologica, Integrata, Conservativa e Biodinamica), selezionabili tramite un elenco a scelta multipla. A questo punto è possibile salvare il lavoro tramite un tasto “Salva” e attendere l’approvazione del prodotto.


Per quanto riguarda invece il caricamento di prodotti trasformati, il caricamento risulta analogo a quello del Trasformatore tranne per il fatto che il Distributore di Tipicità non può collegare i prodotti trasformati alle materie prime di origine, dunque:


Tramite la propria dashboard personale, può caricare i prodotti desiderati specificando: nome, descrizione, immagine/i e prezzo. Successivamente, in un’altra sezione denominata “Informazioni sul prodotto” e composta dalle sottosezioni “Certificazioni” e “Processi di trasformazione” sarà possibile aggiungere le certificazioni, a loro volta suddivise in due sottosezioni: certificazioni di prodotto (DOP, IGP e GlobalG.A.P.) e certificazioni di processo (HACCP, ISO9001, ISO22005) in formato PDF e i processi di trasformazione (Fermentazione, Coagulazione, Molitura, Macinazione e Maturazione), selezionabili tramite un elenco a scelta multipla. A questo punto è possibile salvare il lavoro tramite un tasto “Salva” e attendere l’approvazione del prodotto.

Distributore di Tipicità


Scenarios
Scenario
1.  Il caso d'uso inizia quando il Distributore di Tipicità vuole creare un nuovo pacchetto
2.  Il Distributore di Tipicità naviga alla Sezione Articoli
3.  Il Distributore di Tipicità clicca su "Crea Pacchetto"
4.  Il Distributore di Tipicità clicca su "Aggiungi Articoli"
5.  Il Distributore di Tipicità seleziona i prodotti desiderati dalla lista tramite checkbox
6.  Il Distributore di Tipicità compila il modulo con le informazioni richieste
7.  Il Distributore di Tipicità clicca su "Salva"
8.  SYSTEM Il sistema salva le informazioni del pacchetto
9.  SYSTEM Il sistema invia il pacchetto al Curatore per l'approvazione
10.  SYSTEM Il sistema mostra lo stato "In attesa di approvazione"
11.  SYSTEM Il sistema aggiorna lo stato del pacchetto in base alla decisione del Curatore
12.        Il caso d'uso termina
Extensions
2.a. Annullamento dell'operazione
1.  Il Distributore di Tipicità preme il tasto "Annulla"
2.  SYSTEM Il sistema termina l'operazione di caricamento
3.  SYSTEM Il sistema reindirizza alla sezione Pacchetti
5        .a. Errore nella selezione degli articoli
1.  SYSTEM Il sistema verifica la compatibilità degli articoli
2.  Eventuale blocco di selezioni non consentite
6        .a. Errore durante il caricamento delle informazioni
1.  SYSTEM Il sistema rileva campi obbligatori non compilati
2.  SYSTEM Il sistema mostra un messaggio di errore
3.  SYSTEM Il sistema richiede di completare le informazioni mancanti
4.  Il flusso ritorna al passo 6
6        .b. Caricamento fallito per omonimia di pacchetti
1.  SYSTEM Il sistema rileva un pacchetto con lo stesso nome
2.  SYSTEM Il sistema blocca il caricamento
3.  SYSTEM Il sistema notifica il problema di omonimia
4.  Il Distributore di Tipicità modifica il nome del pacchetto
7        .a. Errore di sistema durante il salvataggio
1.  SYSTEM Il sistema notifica il problema
2.  SYSTEM Il sistema reindirizza alla sezione Pacchetti
10.a. Pacchetto non approvato
1.  SYSTEM Il sistema aggiorna lo stato a "Non approvato"
2.  SYSTEM Il sistema invia una notifica al Distributore di Tipicità
3.  Il caso d'uso termina
